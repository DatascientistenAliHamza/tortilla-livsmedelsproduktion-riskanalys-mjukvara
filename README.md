# Tortilla Production Simulator

## A visual production simulation concept for modern factories

**Tortilla Production Simulator** is an interactive production simulation and visual dashboard concept designed to show how factory data, machine status, production flow and operational events can be presented in a clear, modern and decision-friendly way.

The project demonstrates how a factory can move from static reporting and disconnected production data toward a more visual, real-time and intelligence-driven way of understanding production performance.

Although the example is based on tortilla production, the concept can be applied to many types of industrial environments, including food production, packaging, process industry, automotive manufacturing, pharmaceuticals and large-scale automated production lines.

* Are we producing at the expected rate?
* Where are the bottlenecks?
* What should the team focus on first?
* How can we improve production without guessing?

**Tortilla Production Simulator** shows how production data can be made more understandable, more visual and more useful.

The goal is simple:

The simulator visualizes a production process in a way that makes it easier to understand what is happening in the factory.

The system is designed to show:

* Production flow
* Machine activity
* Process status
* Simulated real-time events
* Operational changes
* Production behavior over time
* Clear visual feedback
* A modern control-room style interface

This creates a foundation for better communication between management, operators, engineers and technical teams.

# Advanced technical overview

This project is built as an interactive Python-based production simulator using Pygame.

The system is structured around a simulation loop where visual objects, machine states, events and dashboard panels are updated continuously.

## Core technical components

* Python application structure
* Pygame-based rendering
* Real-time simulation loop
* Configurable settings
* Visual dashboard layout
* State-based production behavior
* Event messaging
* Animated production elements
* Responsive window handling
* Improved layout logic to reduce overlap
* Slower simulation speed for better readability

## Main technical purpose

The technical goal is to create a visual simulation environment that can represent industrial production behavior in a way that is understandable and extendable.
